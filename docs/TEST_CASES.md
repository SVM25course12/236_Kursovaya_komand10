## Детальные тест-кейсы

### TC-001: Успешная онлайн-запись

**Приоритет:** Высокий

**Предусловия:**
- Сайт запущен
- В БД есть услуги и мастера

**Шаги:**
1. Открыть главную страницу
2. Прокрутить до секции "Онлайн-запись"
3. Заполнить поле "Имя" — "Тест Тестов"
4. Заполнить поле "Телефон" — "+79001234567"
5. Выбрать услугу из списка
6. Дождаться загрузки мастеров
7. Выбрать мастера
8. Выбрать дату (сегодня или позже)
9. Выбрать время
10. Нажать "Записаться"

**Ожидаемый результат:**
- Появляется зелёное сообщение "Запись успешно создана!"
- Форма очищается
- В админке появляется новая запись со статусом "Новая"

---

### TC-002: Валидация формы - пустые поля

**Приоритет:** Высокий

**Шаги:**
1. Открыть форму записи
2. Не заполнять поля
3. Нажать "Записаться"

**Ожидаемый результат:**
- Форма не отправляется
- Браузер показывает валидацию на обязательных полях

---

### TC-003: Валидация - прошедшая дата

**Приоритет:** Средний

**Шаги:**
1. Открыть форму записи
2. Попытаться выбрать вчерашнюю дату

**Ожидаемый результат:**
- Невозможно выбрать прошедшую дату (поле имеет min=сегодня)

---

### TC-004: Динамическая загрузка мастеров

**Приоритет:** Высокий

**Предусловия:**
- Услуга "Женская стрижка" связана с мастером "Анна Петрова"
- Услуга "Маникюр" связана с мастером "Мария Иванова"

**Шаги:**
1. Открыть форму записи
2. Выбрать "Женская стрижка"
3. Проверить список мастеров
4. Выбрать "Маникюр"
5. Проверить список мастеров

**Ожидаемый результат:**
- При выборе стрижки — только "Анна Петрова"
- При выборе маникюра — только "Мария Иванова"

---

### TC-005: Навигация - Intersection Observer

**Приоритет:** Средний

**Шаги:**
1. Открыть сайт на десктопе
2. Медленно прокрутить страницу вниз
3. Наблюдать за боковой навигацией

**Ожидаемый результат:**
- При прокрутке к каждой секции соответствующая точка в навигации становится активной (золотой цвет)

### TC-008: API - создание записи

**Приоритет:** Высокий

**Инструмент:** Postman или curl

**Запрос:**
```bash
curl -X POST http://127.0.0.1:8000/api/appointments/ \
  -H "Content-Type: application/json" \
  -d '{
    "client_name": "API Test",
    "client_phone": "+79001234567",
    "master": 1,
    "service": 1,
    "date": "2025-12-25",
    "time": "14:00"
  }'
```

**Ожидаемый результат:**
- Статус 201 Created
- JSON с success: true
- Запись создана в БД

---

### TC-009: API - валидация времени

**Приоритет:** Средний

**Запрос:**
```bash
curl -X POST http://127.0.0.1:8000/api/appointments/ \
  -H "Content-Type: application/json" \
  -d '{
    "client_name": "Test",
    "client_phone": "+79001234567",
    "master": 1,
    "service": 1,
    "date": "2025-12-25",
    "time": "23:00"
  }'
```

**Ожидаемый результат:**
- Статус 400 Bad Request
- Ошибка: "Запись возможна с 09:00 до 21:00"

---

### TC-010: Админка - изменение статуса записи

**Приоритет:** Высокий

**Шаги:**
1. Войти в /admin/ под admin
2. Перейти в раздел "Записи"
3. Найти запись со статусом "Новая"
4. Изменить статус на "Подтверждена"
5. Сохранить

**Ожидаемый результат:**
- Статус изменён
- Запись отображается с новым статусом

| Поле | Значение |
|------|----------|
| Имя | Иван Петров |
| Телефон | +79001234567 |
| Email | test@example.com |
| Услуга | Женская стрижка (id=1) |
| Мастер | Анна Петрова (id=1) |
| Дата | 2025-12-25 |
| Время | 14:00 |

### Невалидные данные для тестирования

| Сценарий | Телефон |
|----------|---------|
| Слишком короткий | +7900123 |
| Буквы | +7900abc4567 |
| Без + | 89001234567 |
