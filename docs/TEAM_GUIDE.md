# Руководство команды разработки

## Обзор проекта

Сайт салона красоты — учебный проект, демонстрирующий полный цикл разработки веб-приложения. Проект реализуется командой из 6 ролей.

---

## Состав команды и обязанности

### Тимлид

**Основные задачи:**
- Определение архитектуры проекта
- Выбор технологического стека
- Планирование этапов разработки
- Код-ревью и контроль качества
- Координация работы команды

**Принятые решения:**
1. **Архитектура:** Монолитное Django-приложение с REST API
2. **Стек:** Python 3.x + Django 4.2 + SQLite + Vanilla JS
3. **Структура:** Одностраничный лендинг с секциями

**Обоснование решений:**
- Django выбран как зрелый, хорошо документированный фреймворк
- SQLite — для упрощения развёртывания (нет внешних зависимостей)
- Vanilla JS вместо React/Vue — для понятности учебного проекта

---

### Backend-разработчик

**Основные задачи:**
- Реализация моделей Django
- Создание REST API (ViewSets, сериализаторы)
- Настройка авторизации (Django Auth)
- Валидация данных
- Настройка админ-панели

**Реализованные компоненты:**
1. `models.py` — 5 моделей данных
2. `serializers.py` — сериализаторы для API
3. `views.py` — представления и API endpoints
4. `admin.py` — кастомизированная админ-панель

**API Endpoints:**
| Метод | URL | Описание |
|-------|-----|----------|
| GET | /api/services/ | Список услуг |
| GET | /api/masters/ | Список мастеров |
| POST | /api/appointments/ | Создание записи |
| GET | /api/contacts/ | Контакты |
| GET | /api/salon-info/ | Информация о салоне |

---

### Frontend-разработчик

**Основные задачи:**
- Вёрстка HTML-шаблонов
- Создание CSS-стилей
- JavaScript-логика (навигация, формы)
- Адаптивный дизайн
- UI/UX оптимизация

**Реализованные компоненты:**
1. `index.html` — одностраничный лендинг с 6 секциями
2. `style.css` — полный набор стилей (~600 строк)
3. `navigation.js` — Intersection Observer для навигации
4. `booking.js` — форма записи с AJAX

**Использованные технологии:**
- CSS Scroll Snap — для "прилипания" к секциям
- Intersection Observer API — отслеживание видимых секций
- Fetch API — асинхронные запросы к backend
- CSS Grid/Flexbox — адаптивная вёрстка

**Структура страницы:**
```
#hero     → Главный баннер
#about    → О салоне
#services → Услуги и цены
#masters  → Мастера
#booking  → Форма записи
#contacts → Контакты и карта
```

### Базист 

**Основные задачи:**
- Проектирование структуры БД
- Создание ER-диаграммы
- Оптимизация запросов
- Написание миграций
- Создание начальных данных (fixtures)

**Реализованные компоненты:**
1. 5 таблиц в SQLite
2. M2M связь Master-Service
3. Fixtures с демо-данными

**ER-диаграмма:**
```
SalonInfo (1) ─ Главная
Contact (1) ─── Контакты
Service (N) ◄─► Master (M) через MasterService
     │              │
     └─► Appointment ◄─┘
```

**Команды для работы с БД:**
```bash
python manage.py makemigrations  # Создать миграции
python manage.py migrate         # Применить миграции
python manage.py loaddata initial_data  # Загрузить данные
```

**Основные задачи:**
- Написание README.md
- Документирование API
- Описание структуры БД
- Комментирование кода
- Подготовка инструкций

**Созданная документация:**
1. `README.md` — общее описание проекта
2. `docs/API.md` — документация REST API
3. `docs/DATABASE.md` — структура БД
4. `docs/TEST_CASES.md` — тест-кейсы
5. `docs/TEAM_GUIDE.md` — руководство команды

**Стандарты документирования:**
- Docstrings в Python-коде (формат Google)
- Комментарии на русском языке
- Markdown для документации

### Тестировщик

**Основные задачи:**
- Составление чек-листов
- Написание тест-кейсов
- Автоматические тесты (unit, integration)
- Проверка UI/UX
- Регрессионное тестирование

**Реализованные компоненты:**
1. `tests/test_models.py` — unit-тесты моделей
2. `tests/test_api.py` — интеграционные тесты API
3. `docs/TEST_CASES.md` — ручные тест-кейсы

**Запуск тестов:**
```bash
python manage.py test tests  # Все тесты
python manage.py test tests.test_models  # Только модели
python manage.py test tests.test_api     # Только API
```

## Рабочий процесс

### Этапы разработки

1. **Планирование** (Тимлид)
   - Определение требований
   - Выбор технологий
   - Распределение задач

2. **Проектирование БД** (Базист)
   - ER-диаграмма
   - Модели Django
   - Миграции

3. **Backend** (Backend-разработчик)
   - API endpoints
   - Бизнес-логика
   - Админ-панель

4. **Frontend** (Frontend-разработчик)
   - Вёрстка
   - Стили
   - JavaScript

5. **Документация** (Технический писатель)
   - README
   - API docs
   - Комментарии

6. **Тестирование** (Тестировщик)
   - Тест-кейсы
   - Автотесты
   - Баг-репорты

## Полезные команды

```bash
# Запуск сервера разработки
python manage.py runserver

# Создание суперпользователя
python manage.py createsuperuser

# Запуск тестов
python manage.py test

# Проверка кода
python manage.py check

# Сбор статики (для продакшена)
python manage.py collectstatic
```
